以下是抓取到的最新新聞原始資料（JSON）：

```
!`${CLAUDE_SKILL_DIR}/run.sh $ARGUMENTS`
```

---

根據上方 JSON 資料，以**繁體中文**整理成一份新聞摘要報告，格式如下：

## 📰 News Digest — {今日日期} （近 {時間窗}）

依 `category` 分區，每區用對應標題。**只輸出資料中實際出現的分區**：

### 🤖 AI / LLM
### 🔐 資安 Security
### 🟠 Hacker News
### 💻 科技 Tech

每一則新聞用以下格式（同分區內依重要性排序，最多 8 則）：

- **[{標題翻成精煉繁中}]({link})** — {一句話繁中摘要，點出重點/影響}（來源：{source}）
  原文連結：{link}

其中 `{link}` 一律使用 JSON 中該則新聞的 `link` 欄位原值，**不可省略、不可改寫、不可虛構**。

---

> 輸出風格要求：
> - **每一則新聞都必須附上可點擊的原文連結**：標題用 Markdown 連結 `[標題]({link})`，並在下一行以「原文連結：{link}」附上完整網址，方便在終端機直接點開。`{link}` 必須與 JSON 中的 `link` 欄位完全一致。
> - 「原文連結：{link}」這一行**必須縮排兩個空格**，使其對齊到上方 bullet 的文字下方（成為該 bullet 的續行），不可頂格。
> - 標題與摘要都用**繁體中文**，但保留專有名詞原文（如 GPT、CVE 編號、公司/產品名）。
> - 摘要要說「發生什麼 + 為何重要」，不要只翻譯標題。
> - 同一事件被多個來源報導時，合併成一則並標註多個來源，同時保留各來源的原文連結。
> - 內容空泛、純廣告、純電子報導引（如 summary 為空白）的項目可略過。
> - 若某分區無資料則整區略過。
> - 全部整理完後，最末加一行：**共 {N} 則 · 來源 {M} 個 · 抓取時間窗 {時間窗}**。

> 若 JSON 為空陣列 `[]`，回覆：「此時間窗內沒有抓到新聞，試試放寬時間窗（如 `/news all 72h`）或檢查網路。」並列出 stderr 中的任何 `warn:` 行（哪些 feed 抓取失敗）。
